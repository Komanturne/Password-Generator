# Hexadecimal Password Generator
A simple password generator using Hexadecimal

|                |One Round                      |Two Rounds                  |
|----------------|-------------------------------|-----------------------------|
|EFF Dice (3)        |>20 years `b65c9604db2aeb748743ec0ec3f4fd5df48e801a6ba25c654b946b529b`|>100 years ` b60dc40c9e66ba417881bf25b3f4cb17cd35402756b030744d5378949821c2a4b312b2f1f4baa6f52bb78918e74a7e84c211db23d9274688e568`|
|Random (regular password)          |~10 years `a45c861cd83dfc71`   |>15 years    `b50fc40c9f66bb74782bbe2ab0f3cb1c`        |

You should probably use this with a password manager, and you should probably edit it in order to add capital letters, symbols (#$%^&*!₺), and other things.
You can also edit the hex block (this uses a 512 byte hex block, but can work with 6,144 bytes from experience, though hypothetically could work with more) but it only affects longer text.
Here are some examples of possible passwords:

 - Password1! ->  `845c861cd83dfc712d44` -> `84@#.5c861cd83dfc712d44#!`
 - password ->  `a45c861cd83dfc71` -> `{}845c86**_1cd83df())c712d44.:.`
 - lemurpasscode! -> `b8609c22d52ceb846f82f22ab3b1` -> `b#8609c."22d52ceb8")46f82f22ab3b1(.`

I recommend you to grab two or three words from the EFF dice list, preferably from the long list.

 - envoy_answer,wheat! -> `/B15:58308De0de{}b676F76ec14\fafFf068f4784D!()`
 - yahoo & zookeeper + taekwondo - aesthetic -> `*ad5ca10!,8d07ca8.35767ef21bb3e/\d08680744431a{}()67ae656a397f70449{15ddae0b4c1fd02a}}(4e5ebed7e))!`

You should probably use a password manager, as these passwords are nearly unmemorizable. Something like KeePassXC, or Bitwarden for the browser, is preferable over Closed-Source Managers.
Compared to 'Pseudo-Random' Generation:

 - `Rl15dv45,F#Ac]iel{£Ii*` (Lastpass)
 - `bb5e8f1cd23df56e8277f929c7eefd6cf1` (This program)
 - `a[6*&5N6#930&!98!6£` (Edited from this program)

It is important, especially for shorter passcodes generated by this, that you add symbols and uppercase letters.
This program can also be reverse engineered quite easily, and so it is probably not a good idea to put in something like "password" or "qwerty," especially if you do not edit it afterwards.
This program also suffers from the predictability of hexadecimal as a format. So it is better to stick to passwords that are above 18 characters. 
It also serves the problem of not being a cryptographic hash, whereas adding more to it does not affect the end result. So it is better to not necessarily use this hexadecimal key. There are many places online where you can convert text to hexadecimal, and you can use random characters to create a stronger code.
